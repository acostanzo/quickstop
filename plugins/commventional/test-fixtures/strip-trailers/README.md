# Strip-trailers fixtures

Regression suite proving that commventional v2.0's `bin/strip-trailers.sh`
produces byte-identical output to the v1.x `hooks/enforce-ownership.sh`
perl substitution chain — invariant C of the v2.0 migration ticket
(`project/tickets/closed/phase-2-commventional-adr-006-conformance.md`).

## Files

| Path | Purpose |
|---|---|
| `cases.json` | The fixture set — `[{name, input, expected}]`. The `expected` field for each case was derived once, at fixture-creation time, by piping the input through the v1.x perl chain. Committed verbatim; this is the regression bar. |
| `cases.test.sh` | Pipes each case's input through `bin/strip-trailers.sh` and asserts byte-equivalence with `expected`. Exit non-zero on any divergence. |
| `generate-cases.sh` | Regenerates `cases.json` by running the canonical v1.x perl chain. Run when adding new fixture cases. The chain itself must not be edited — drift between this and `bin/strip-trailers.sh` is what the test detects. |

## Cases

| # | Name | What it covers |
|---|---|---|
| 1 | `single-coauthor` | Single `Co-Authored-By:` trailer on its own line |
| 2 | `multi-coauthor` | Multiple `Co-Authored-By:` trailers stacked |
| 3 | `generated-with-claude-code` | `🤖 Generated with Claude Code` footer |
| 4 | `generated-by-claude` | `Generated by Claude` footer (alt phrasing) |
| 5 | `trailer-and-footer` | Both patterns in one message |
| 6 | `lowercase-coauthor` | `co-authored-by:` (lowercase) — case-insensitive match |
| 7 | `pr-body-markdown` | PR-body shape with patterns embedded in markdown structure |
| 8 | `already-clean` | Input with no patterns — idempotent passthrough |
| 9 | `crlf-line-endings` | CRLF input — documents the v1.x behaviour (does not assert a particular newline normalisation; the v1.x chain leaves the `\r` in place when no `\n`-bounded match removes it) |

## Run

```bash
./cases.test.sh
```

Exits 0 on all-green, non-zero on any failing case. Failure output dumps
the divergent expected vs actual to stderr.

## Adding a case

1. Add an `input_<slug>` variable and an `emit_case` line to `generate-cases.sh`.
2. Re-run `generate-cases.sh` — it overwrites `cases.json`.
3. Re-run `cases.test.sh` — must still pass.
4. Commit `cases.json` and the generator change together.

Never hand-edit `cases.json`. The expected values are the canonical v1.x
chain's output; rederive via the generator instead.
