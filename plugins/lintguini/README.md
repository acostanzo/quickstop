# Lintguini

Lint-posture toolkit for Claude Code. Lintguini owns a consumer repo's lint configuration the way avanti owns its `project/` tree: configure, lint, format, and fix are the daily surface; a Pronto-sibling audit for lint-posture ships alongside. Every config lintguini writes is a mechanical projection of the rubric at [`plugins/pronto/references/roll-your-own/lint-posture.md`](../pronto/references/roll-your-own/lint-posture.md) — the rubric is the source of truth, lintguini's templates are derivatives, and ADR-008 pins the contract.

## Quick start

```bash
/plugin install lintguini@quickstop

/lintguini:configure --language python --strict   # write rubric-aligned config
/lintguini:lint                                    # run linters, surface findings
/lintguini:format --check                          # diff-only formatter run
```

## Skills

| Skill | Purpose |
|---|---|
| [`/lintguini:configure`](skills/configure/SKILL.md) | Detect language(s), write canonical lint/format configs into the consumer repo. `--strict` / `--lenient` / `--minimal` pick the strictness band; `--ci` also wires a lint step into the detected CI surface. Idempotent. |
| [`/lintguini:lint`](skills/lint/SKILL.md) | Run the configured linter(s) and surface findings as `path:line:rule:message` (one per line). `--language <lang>` scopes to a single language in polyglot repos. |
| [`/lintguini:format`](skills/format/SKILL.md) | Run the configured formatter(s). `--check` reports diffs without writing and exits non-zero when diffs exist (CI-friendly); default applies them. |
| [`/lintguini:fix`](skills/fix/SKILL.md) | Auto-fix wrapper. `--apply` performs safe fixes (formatting, import sort, trivial rule auto-fixes); `--apply-semantic` emits a unified diff for behaviour-changing fixes without writing. Default is read-only. |
| [`/lintguini:audit`](skills/audit/SKILL.md) | Pronto-sibling audit for the `lint-posture` rubric dimension. Pure shell + grep + awk + jq, no LLM dispatch. |

## Rubric authority

The rubric at [`plugins/pronto/references/roll-your-own/lint-posture.md`](../pronto/references/roll-your-own/lint-posture.md) is the canonical definition of "good lint posture" per language. Lintguini's templates are mechanical projections of that rubric — when the two disagree, the rubric wins; the template is the bug. The contract is pinned in [ADR-008](../../project/adrs/008-lintguini-rubric-authority.md):

1. **Source of truth is the rubric.** Templates derive from it; they have no independent strictness definitions.
2. **New strictness bands land in the rubric first.** `--strict | --lenient | --minimal` are bounded variations *within* rubric-defined bands.
3. **New languages land in the rubric first.** Adding a language to lintguini without a rubric section is a violation.
4. **Generated configs are self-describing.** Every config lintguini writes leads with a `# Generated by lintguini — see plugins/pronto/references/roll-your-own/lint-posture.md for the source of truth.` comment.
5. **The audit grades against the rubric, not against templates.** A repo configured by `/lintguini:configure --strict` will score at the rubric's strict band on `/lintguini:audit` by construction.

## Languages

Lintguini covers exactly the rubric's language set. Each language uses the ecosystem's conventional config shape — no invented file shapes.

| Language | Linter config | Formatter config | Tool dispatch |
|---|---|---|---|
| Python | `pyproject.toml` (`[tool.ruff]`, `[tool.ruff.lint]`) | `pyproject.toml` (`[tool.ruff.format]`) | `ruff` |
| JavaScript | `biome.json` or `.eslintrc*` + `.prettierrc` | same file | `biome` (or eslint+prettier) |
| TypeScript | `biome.json` or `eslint.config.*` + `.prettierrc` | same file | `biome` (or eslint+prettier) |
| Rust | `Cargo.toml` (`[lints.rust]`, `[lints.clippy]`) | `rustfmt.toml` | `cargo clippy` + `rustfmt` |
| Ruby | `.rubocop.yml` or `standard.yml` | same file (autocorrect) | `rubocop` or `standardrb` |
| Go | `.golangci.yml` | `gofmt` (no config file) | `golangci-lint` + `gofmt` |

## /lintguini:configure — write canonical configs

Detects the language(s) in the consumer repo via [`bin/lintguini-detect-language.sh`](bin/lintguini-detect-language.sh), picks the strictness band per `--strict` / `--lenient` / `--minimal` (default `--strict`), and writes the matching template into the conventional path for each tool. The first non-blank line of every generated config is the lintguini provenance comment.

```bash
/lintguini:configure --language python --strict
/lintguini:configure --language ruby --strict --ruby-tool standardrb
/lintguini:configure --strict --ci                       # all detected languages + CI lint step
```

Idempotent — running configure twice on a config'd repo writes byte-equivalent output and is a no-op against the working tree. When config already exists and diverges from the rubric baseline, configure surfaces the diff and asks before overwriting (no silent rewrites).

`--ci` detects the consumer's CI surface (`.github/workflows/*.yml`, `.gitlab-ci.yml`, `Makefile`, `lefthook.yml`, `.pre-commit-config.yaml`) and adds a lint step targeting the most-conventional surface for the detected language. Without `--ci`, configure writes only the local config and leaves CI alone — repos that prefer to wire CI by hand keep that ergonomics.

## /lintguini:lint — run linters, surface findings

Thin wrapper over [`bin/lintguini-lint.sh`](bin/lintguini-lint.sh). Detects the configured tools (gated on the lintguini provenance marker), dispatches them, and emits findings in a single locked shape:

```
path:line:rule:message
```

One finding per line. The shape is pinned because downstream tooling depends on it — most notably `score-lint-pass-rate.sh` (the M5 conditional audit scorer) which parses this output to grade pass rate.

```bash
/lintguini:lint                       # all configured languages
/lintguini:lint --language typescript # scope to one language
```

When no lintguini-managed config exists for a language, lint emits an empty-scope message to stderr (`<lang>: not configured (run /lintguini:configure --language <lang>)`) and exits 0 for that language. Polyglot output sections by language with `# python — N findings` headers; single-language runs emit no header. See [`skills/lint/SKILL.md`](skills/lint/SKILL.md) for the locked exit-code map.

## /lintguini:format — run formatters

Thin wrapper over [`bin/lintguini-format.sh`](bin/lintguini-format.sh). Same gating as lint (lintguini-managed config marker required). `--check` is the CI-friendly mode: reports diffs without writing, exits non-zero when diffs exist. Default applies fixes in place.

```bash
/lintguini:format               # rewrite files in place
/lintguini:format --check       # diff-only, exits non-zero on drift
```

See [`skills/format/SKILL.md`](skills/format/SKILL.md).

## /lintguini:fix — auto-fix wrapper

Three modes, mirroring `/inkwell:tidy`'s mechanical-vs-semantic split:

| Mode | What it does | Writes? |
|---|---|---|
| (default) | Read-only. Reports what `--apply` and `--apply-semantic` would do | No |
| `--apply` | Safe fixes only — formatting, import sort, trivial rule auto-fixes the linter flags as safe | Yes (working tree) |
| `--apply-semantic` | Emits a unified diff for behaviour-changing fixes (e.g. dead-code removal, exception-handling rewrites) | No |

The split is load-bearing: mechanical changes have a single correct answer and are silent; semantic changes always come with a diff so they're reviewed, not trusted. See [`skills/fix/SKILL.md`](skills/fix/SKILL.md).

## /lintguini:audit — Pronto sibling

Lintguini ships as a Pronto sibling for the **`lint-posture`** dimension (weight 15 in the readiness rubric). The audit stays deterministic — pure shell + grep + awk + jq, no LLM dispatch in scorers — and runs orthogonally to the writer surface.

### Standalone invocation

```bash
/lintguini:audit --json
```

Emits a v2 wire-contract JSON envelope to stdout. The `observations[]` field carries entries pronto's rubric translates into a dimension score; `composite_score` is `null` because the rubric path is the sole authority on dimension scoring.

### Scorers

[`bin/build-envelope.sh`](bin/build-envelope.sh) dispatches the scorers in fixed order and slurps their non-empty stdouts into `observations[]`.

Always-on scorers (4):

- [`score-linter-presence.sh`](scorers/score-linter-presence.sh) — `linter-strictness-ratio` against the rubric's per-language baseline.
- [`score-formatter-presence.sh`](scorers/score-formatter-presence.sh) — `formatter-configured-count` (boolean dressed as a count).
- [`score-ci-lint-wired.sh`](scorers/score-ci-lint-wired.sh) — `ci-lint-wired-ratio` over detected CI surfaces.
- [`score-suppression-count.sh`](scorers/score-suppression-count.sh) — `lint-suppression-count` of `# noqa` / `// eslint-disable` / `#[allow(...)]` / `// nolint:` / `# rubocop:disable` markers.

Conditional scorers (gated on the lintguini provenance marker — emit empty-scope on non-lintguini consumers, preserving today's audit semantics):

- [`score-lint-pass-rate.sh`](scorers/score-lint-pass-rate.sh) — actually runs the configured linter against the consumer repo and grades the pass rate (`files_clean / files_scanned`). Polyglot-aggregated; per-language slug list in evidence.

A `score-suppression-staleness.sh` (flag suppressions for rules that no longer fire) is a documented follow-up — the comment-out-and-rerun heuristic is approximate enough that the "ship if cheap" bar wasn't met for 0.5.0.

### Pronto handshake

`plugin.json` declares `compatible_pronto: ">=0.2.0"`. Pronto checks this at dispatch time — if the installed pronto is outside the declared range, pronto skips the sibling audit and scores the dimension by presence only.

## Architecture

- **Skills.** Five `SKILL.md` files under [`skills/`](skills/) — `configure`, `lint`, `format`, `fix`, `audit`.
- **Bin scripts.** Workhorses under [`bin/`](bin/) — `build-envelope.sh` (audit orchestrator), `lintguini-detect-language.sh`, `lintguini-configure.sh`, `lintguini-lint.sh`, `lintguini-format.sh`, `lintguini-fix.sh`.
- **Templates.** Per-language config templates under [`templates/`](templates/), one subdirectory per supported language with `strict` / `lenient` / `minimal` band variants. Mechanical projections of the rubric — the lone source of truth.
- **Scorers.** Five deterministic shell scorers under [`scorers/`](scorers/): four legacy + one M5 conditional (lint-pass-rate). Each emits at most one v2-envelope observation.
- **Provenance marker.** Every config lintguini writes carries a `Generated by lintguini` leading comment (or `_provenance` JSON key for biome.json). Lint, format, and fix gate on this marker for idempotency; the M5 audit scorer gates on it for the conditional A4 invariant.

The plugin ships no commands, no agents, no hooks, no MCP servers. ADR-006 §2 (no silent mutation of consumer artefacts) and §3 (vacuously satisfied: lintguini ships no hooks) hold across the surface — `/lintguini:configure` mutates the consumer's config files, but only when invoked explicitly with confirmation on divergent existing config.
