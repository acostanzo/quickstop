## 18:49

Avanti workflow polish (PR forthcoming). Two issues surfaced by the
inkwell-expansion M-stage dispatches — fixed in one PR, version bump
to 0.1.4.

- A2: removed `disable-model-invocation: true` from `/avanti:promote`,
  `/avanti:pulse`, `/avanti:ticket`. The flag was set defensively to
  keep these as "explicit user gestures," but it conflated autonomous
  model action with directed dispatch (Alfred → dev-chat → skill) and
  also broke promote's own Phase 4 chain into pulse. The interactive
  `AskUserQuestion` calls preserve human-in-the-loop semantics
  regardless of caller; the state-machine guards (active → done
  requires all tickets closed; ADR supersession requires
  `--supersedes`) are enforced by the skill body, not by the
  invocation gate. Precedent: `/avanti:audit` had the flag removed
  in Phase 1.5 T4 under the same reasoning. The flag stays untouched
  on `/avanti:plan`, `/avanti:adr`, `/avanti:status` — those weren't
  surfaced as friction points and are candidates for a follow-up
  sweep if dispatch use cases emerge.
- A3: `/avanti:ticket`, `/avanti:plan`, `/avanti:adr`, `/avanti:pulse`
  now `mkdir -p` their deterministic destination subdirectories
  (`tickets/open/`, `plans/active/`, `adrs/`, `pulse/`) when missing,
  rather than aborting with a `/pronto:init` pointer. The parent
  `project/` container is still pronto's to scaffold; if the entire
  subtree is missing, the skills fall back to a softer "you may also
  want `/pronto:init`" note in the report instead of blocking the
  write. `/avanti:promote` similarly `mkdir -p`s `plans/done/` and
  `tickets/closed/` before moving artefacts so first-promotion-of-a-
  state-folder can't fail on a missing destination.

## 19:05

Released inkwell 0.4.1: four follow-up patches bundled.

- I1: `inkwell-tidy.sh` now sources `scorers/_common.sh` for the
  shingle/Jaccard math. Tidy and `score-duplicate-density.sh` cannot
  diverge on the algorithm — one helper, two callers. Tidy's local
  `bigrams_for` and `jaccard_files` are gone; only the per-pair
  `ge`/`lt` float comparators remain inline.
- I2: portable `realpath` shim. The link-rewriter's
  `realpath -m --relative-to=` and `realpath -m` calls are GNU
  coreutils only; macOS `realpath` would fail silently. New
  `_path_canonical` / `_path_relative_to` helpers probe for the GNU
  flag set once, fall back to a Python 3 shim on BSD/macOS. Mirrors
  the GNU/BSD `stat` detection in `inkwell-index.sh`. New test case
  exercises the rename-rewrite path on a controlled fixture.
- I3: `/inkwell:doc` now scaffolds new docs under
  `docs/<template-dir>/<slug>.md` — `concepts/`, `howtos/`,
  `reference/`, `tutorials/`, matching the existing tree convention.
  SKILL.md §2b carries the singular-template → plural-directory
  mapping inline. Resolver continues to scan `docs/**/*.md`
  recursively, so existing docs at any depth are still matched on
  the update path.
- I4: quieter `## Related` placeholder. Templates ship with
  `<!-- inkwell:related — populate when the doc earns siblings -->`
  instead of the bare `-`. Tidy and `score-backlink-coverage.sh`
  treat the comment as writer-acknowledged-empty (no
  `missing-related` finding on fresh scaffolds). The bare `-` still
  fires so legacy docs get nudged to migrate.

  **Design choice — Option A (placeholder shape) over Option B
  (mute by mtime).** Option A makes the writer's intent explicit
  and keeps tidy time-independent — the determinism guarantee
  (triple-run ⇒ identical stdout) is the same regardless of when
  tidy runs against a tree. Option B would have introduced
  mtime-based windowing that complicates the deterministic
  contract.

Two new test cases in `inkwell-tidy.test.sh`: rename-rewrite
exercise (I2 shim) and comment-vs-bare-dash placeholder behaviour
(I4 Option A). Full suite (`scorers/tests/run-all.sh`) green.
