# Pulse — 2026-05-06

## 13:30

Released commventional 2.1.0 — inline review comments via grouped review submission.

- Reviews used to glom every finding into a single `gh pr comment` wall-of-text. Real reviewers leave inline comments on specific lines under one review submission with a short overall verdict; v2.1 makes that the path of least resistance by locking a JSON contract between the formatter (LLM) and the poster (deterministic bash).
- `agents/review-formatter.md` now emits a single JSON document — `{ verdict: { event, body }, comments: [{ path, line, side, label, subject, discussion? }, ...] }` — instead of human-readable text. Output spec is JSON-only: no prose, no fences, no commentary. The poster pipes the agent's stdout straight into `jq`.
- New `bin/commventional-post-review.sh` consumes the JSON on stdin, validates shape (path/line/label/subject required per comment; line must be numeric; event ∈ {COMMENT, APPROVE, REQUEST_CHANGES}; side ∈ {LEFT, RIGHT}), resolves head SHA via `gh pr view`, builds the GitHub review-API payload, and submits via a single `gh api POST repos/{owner}/{repo}/pulls/{n}/reviews --input -`. One review, N inline comments grouped under it.
- `skills/commventional/SKILL.md` "Code Reviews" step 3 rewritten and a "Response contract — locked" subsection added inline so a future contributor can wire against the field shape without re-deciding.
- Test surface: `test-fixtures/post-review/` ships `sample-review.json` (full review, 3 comments), `sample-review-missing-path.json` (validation rejection), `sample-review-empty.json` (verdict-only). `post-review-dryrun.test.sh` covers all four cases (rendering, rejection, empty, triple-run determinism). `--head-sha` flag bypasses live `gh pr view` so the test surface runs without network or gh auth. New top-level runner at `tests/run-all.sh` walks the existing trailer-stripping suites plus the new post-review test.
- Smoke pass against PR #102 produced the expected `gh api` invocation referencing `repos/acostanzo/quickstop/pulls/102/reviews` with a head SHA matching `gh pr view 102 --json headRefOid`. Comment body preserves the conventional-comments shape on the wire (`label: subject\n\ndiscussion`).
- Out of scope: APPROVE/REQUEST_CHANGES heuristics (the agent decides; the poster passes through), `commit-crafter` changes, the trailer-stripping skills, `user-invocable: false` on the auto-skill.
