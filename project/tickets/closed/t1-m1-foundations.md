# t1 — M1 foundations + parse-lintguini removal

## Context

The first milestone of `lintguini-expansion`. Lands the per-language config templates that every later milestone projects from, the shared language-detection helper that the M2 configure skill and the M5 conditional scorers will both consume, and a scoped README delta that introduces the toolkit shape without committing to the full rewrite (deferred until after M3, when the writer/runner surface is real). In the same milestone — separate atomic commit — strips the deprecated `parse-lintguini` transitional agent so the deck is clear before the new surface lands. Mirrors the M1 pattern from `inkwell-expansion` (T1 there did the same template + `parse-inkwell` strip).

Implements the "Bin layout (post-M5)" template directories, the "What gets deleted" section, and the M1 README delta described under "The model" in `project/plans/active/lintguini-expansion.md`. Constrained by ADR-008 — the templates are mechanical projections of the rubric at `plugins/pronto/references/roll-your-own/lint-posture.md`, not parallel definitions.

## Acceptance criteria

- `plugins/lintguini/templates/python/`, `templates/javascript/`, `templates/typescript/`, `templates/rust/`, `templates/ruby/`, `templates/go/` all exist; each holds the canonical config file(s) for that ecosystem with `--strict`, `--lenient`, and `--minimal` variants derived from the rubric.
- Every template's first non-blank line is the self-describing comment from ADR-008: `Generated by lintguini — see plugins/pronto/references/roll-your-own/lint-posture.md for the source of truth.` (Comment syntax matches the file's format — `#` for TOML/YAML, `//` for JSON/JS, etc.)
- `plugins/lintguini/bin/lintguini-detect-language.sh` exists, is executable, and prints the detected language(s) one per line for a fixture repo (covers Python, JavaScript, TypeScript, Rust, Ruby, Go).
- `plugins/lintguini/scorers/_common.sh` exists and is sourced by the four existing scorers (`score-linter-presence.sh`, `score-formatter-presence.sh`, `score-ci-lint-wired.sh`, `score-suppression-count.sh`); the audit envelope on existing fixtures is byte-equivalent to the pre-refactor envelope.
- `plugins/lintguini/README.md` is updated with the M1 delta: a one-paragraph role section announcing the toolkit shape and a one-line surface enumeration (audit + configure + lint + format + fix). Full README rewrite is explicitly deferred — note this in the README itself.
- `plugins/lintguini/agents/parse-lintguini.md` is deleted.
- `grep -r 'parse-lintguini' plugins/` returns zero hits in shipped plugin files (excluding tests' historical fixtures, if any).
- `plugins/pronto/references/recommendations.json` parses as valid JSON; the `lint-posture.parser_agent` field reads `null` (currently does — verify and leave alone).
- The parse-lintguini removal lands as its own atomic commit, separate from the templates / detection / README commits.

## Notes

The detection helper is consumed by both the M2 configure skill (to decide which template to apply) and the M5 conditional scorers (to gate empty-scope behaviour). Putting it in `bin/` makes it executable from skills; mirroring the logic into `scorers/_common.sh` lets the existing shell scorers keep their `source` pattern without requiring a subprocess.

Strict / lenient / minimal bands are bounded variations within the rubric per ADR-008 §1.2. If the rubric covers only "strict" and "loose" for a language, "minimal" is presence-only with the smallest opinion footprint that satisfies pronto's presence check; that's still a rubric-bounded projection, not an extension.
